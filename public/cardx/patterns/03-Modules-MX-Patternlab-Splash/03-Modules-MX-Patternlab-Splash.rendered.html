Welcome to the CardX Design System

A design system is a collection of reusable UI components, guided by clear standards, that can be assembled together to build any number of applications.

We use a customized version of the Atomic Design methodology which weâ€™ve adapted to better suit this project.

There are four distinct levels in this design system:

  • Elements are the basic building blocks. They are our HTML tags such as a form label, an input field, or a button. Elements can also include more abstract elements like color palettes, fonts and even more invisible aspects of an interface like animations.
  • Components are groups of elements combined together. For example, a form label, an input field, and a button can be combined to make a search form component.
  • Modules are groups of components joined together to form a full-width section of the page. For example, a header module might consist of diverse components like a logo, primary navigation, and search form. Building up from elements to components encourages creating standalone, portable, reusable modules.
  • Templates are a linear listing of modules on a page that dictate how an interface looks and behaves. It's here where we see all the requisite elements, components, and modules come together to form a complete layout.

We've created this design system with several goals in mind:

  • Document the patterns using a shared vocabulary
  • Ensure consistency in how we use UI patterns and elements
  • Provide guidance on correct usage of the patterns
  • Encourage re-use of code to reduce redundancy and risk

This design system site is a living document and should be updated when the code underlying the feature changes or when new patterns are introduced.
